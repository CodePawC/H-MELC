# 医院医学装备全生命周期闭环管理平台范围边界

## 项目定位

医院医学装备全生命周期闭环管理平台（Hospital Medical Equipment Lifecycle Closed-loop Management Platform，H-MELC / Equipment OS）是医院医学装备全生命周期运营管理系统，负责具体设备从建档、使用、扫码、报修、维修、保养、巡检、计量、采购、供应商协同、发票、应付、付款、审批到报废的业务闭环。

H-UMDG（Hospital Unified Master Data Governance Platform，医院统一主数据治理平台）是另一个系统，负责基础主数据、标准字典、编码映射、导入清洗、审核发布和外部查询 API。

## 一句话边界

H-UMDG 决定“设备叫什么、归哪类、监管属性是什么、厂家/供应商是谁”。

医院医学装备全生命周期闭环管理平台决定“这台设备在哪里、谁用、怎么修、怎么保养、怎么计量、谁供货、开什么票、付多少钱”。

## 本系统负责

- 设备资产、设备档案、一机一码和扫码解析。
- 报修维修、外修返厂、PM、巡检、计量检测。
- 采购竞价、供应商门户、资质上传、报价协同。
- 发票上传、发票审核、应付款台账、付款登记、付款分摊、账龄分析。
- 工作流审批、权限、审计日志、运营驾驶舱。
- 保存 H-UMDG 返回的引用编码和必要快照。
- H-UMDG 不可用时提供最近一次成功同步的只读缓存。
- 找不到标准名称或厂商机构时提交候选申请，等待 H-UMDG 审核。

## 本系统不负责

- 医疗器械分类目录权威维护。
- 设备分类标准权威维护。
- 设备标准名称权威维护。
- 厂商机构标准身份权威维护。
- 科室、人员、医保耗材目录等全院基础主数据权威维护。
- H-UMDG 的导入清洗、审核发布、编码映射和主数据外部查询 API 治理。

以上由 H-UMDG 负责。
