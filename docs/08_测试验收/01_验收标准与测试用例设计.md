# 《验收标准与测试用例设计 V1.0》
## 一、总体验收原则
系统上线前必须验证四类内容：
| 类型   | 验收重点              |
|------|-------------------|
| 业务闭环 | 流程能不能完整跑通         |
| 数据准确 | 台账、工单、发票、付款金额是否一致 |
| 权限安全 | 谁能看、谁能改、谁能审批      |
| AI可控 | AI是否只辅助、不越权、不直接决策 |


---
## 二、一期核心验收范围

验收颗粒度按 **`docs/00_项目总览/06_文档颗粒度与对齐矩阵.md`** 执行：每个一期模块至少能追溯到需求来源、API 契约、PC 管理端实现状态与测试用例；未落地能力须标明二期、占位或待补专项用例。

| 模块      | 是否一期验收 |
|---------|--------|
| 用户登录与权限 | 是      |
| 设备台账    | 是      |
| 一机一码    | 是      |
| 统一报修中心（含扫码、AI、语音、图片、多渠道入口） | 是      |
| 工程师接单   | 是      |
| PC管理端报修列表/详情与运维动作 | 是（接口 + 管理端：`docs/05/03`） |
| 维修闭环    | 是      |
| 文件上传    | 是      |
| AI维修报告  | 是      |
| 知识库基础问答 | 是      |
| 知识库文档上传（管理端） | 是（接口 multipart，见 **`docs/06` 第七章**） |
| 审计日志    | 是      |
| 供应商门户   | 二期     |
| 发票付款    | 二期     |
| ROI高级分析 | 三期     |

### 自动化契约（工程约束）

后端 **`backend/tests/`** 中与 **`docs/06_接口设计/01_API接口设计.md`** 对齐的用例，常在文件名或注释中标含 **`contract`** / **`doc`**。接口变更须同步或显式 **`skip`**／TODO，避免文档与实现静默偏离。

## 四、统一报修中心测试用例
| 编号         | 场景      | 预期结果      |
|------------|---------|-----------|
| REPAIR-001 | 科室扫码报修 | 设备二维码先进入设备详情页，点击一键报修后进入统一报修中心，自动带出设备信息 |
| REPAIR-002 | 电脑端后台报修 | 设备科可代录报修，先形成统一报修消息，确认后生成工单 |
| REPAIR-003 | 手机端登录报修 | 自动带出登录用户、科室和联系电话，支持附件 |
| REPAIR-004 | 微信公众号报修 | 文本消息写入 `unified_repair_message`，保留渠道与发送人信息 |
| REPAIR-005 | 企业微信/飞书/钉钉机器人报修 | 机器人回调消息标准化入池，回复识别确认卡片 |
| REPAIR-006 | 系统内AI聊天框报修 | AI抽取科室、位置、设备、故障、紧急程度，并追问缺失信息 |
| REPAIR-007 | 语音报修 | 保存语音原文件，生成并保留 `transcribed_text`，再进入AI识别 |
| REPAIR-008 | 图片报修 | 图片附件保存成功，AI可结合图片摘要和文字描述识别故障 |
| REPAIR-009 | 高置信度设备匹配 | 用户确认后生成标准 `repair_order` |
| REPAIR-010 | 中置信度设备匹配 | 展示疑似设备列表，选择后生成工单 |
| REPAIR-011 | 低置信度设备匹配 | 不直接生成工单，进入待确认报修池 |
| REPAIR-012 | 急救/生命支持设备报修 | 即使信息不完整，也自动升级优先级并推送值班人员 |
| REPAIR-013 | 系统预警自动生成报修 | 预警事件形成统一报修消息，按规则进入待确认或自动转工单 |
| REPAIR-014 | 报修进度查询 | AI聊天框或机器人能返回本人/本科室最近工单状态 |
| REPAIR-015 | 渠道人工确认规则 | 指定渠道关闭自动生成后，只能进入待确认池 |
| REPAIR-016 | 重复报修风险 | 同一设备短期多次报修触发风险预警 |
| REPAIR-017 | 无权限扫码 | 只能查看允许内容，不泄露设备或维修敏感信息 |
| REPAIR-018 | 二维码边界 | 二维码作为设备身份入口，不被设计为孤立报修二维码 |

## 九、性能验收指标
| 指标      | 建议标准          |
|---------|---------------|
| 普通页面响应  | ≤ 2秒          |
| 设备列表查询  | ≤ 3秒          |
| 扫码打开页面  | ≤ 2秒          |
| 文件上传    | 支持单文件 ≥ 200MB |
| 并发在线用户  | 一期不少于300      |
| 工单提交成功率 | ≥ 99.5%       |
| AI异步任务  | 可排队、可重试       |
