| 框架 | React | 19 | 
| 构建 | Vite | 8 | 
| 语言 | TypeScript | 6 | 
| 大小 | ~45 KB gzip | — | 轻量，仅扫码+标签打印 |

### 与 H-UMDG 前端技术栈对比

| 维度 | H-MELC | H-UMDG |
|------|--------|--------|
| UI 组件库 | Ant Design 5 + Pro Components | Ant Design 6 |
| 路由 | react-router-dom 7 | 内置 SPA 路由 |
| 状态管理 | Zustand | React hooks |
| HTTP | axios | fetch 封装 |
| 图表/3D | ECharts + three.js | 无 |
| 动画 | framer-motion | 无 |
| CSS | CSS + ProLayout | 全局 CSS |
| Mock | 内置 Mock 系统（`mock/`） | 无 |

> **原则**：两个项目独立部署、独立技术选型。H-MELC 功能丰富（图表/动画/3D/复杂路由），H-UMDG 轻量专注主数据治理。统一身份方案完成后，H-UMDG 作为 SSO 门户承载导航功能。

### 模块边界规则

| 目录 | 可以 | 不可以 |
|------|------|--------|
| `api/` | 导入 `lib/`、`types/` | 导入 React 组件、操作 DOM |
| `lib/` | 导入 `types/`、axios | 导入 React、页面组件 |
| `stores/` | 导入 zustand、`types/` | 直接修改 DOM |
| `pages/` | 导入 `api/`、`components/`、`stores/` | 封装通用逻辑（提取到 `lib/`） |
| `components/` | 导入 `api/`、`lib/`、`types/` | 依赖特定页面路由参数 |
| `navigation/` | 导入 `types/` | 直接调用 API |
| `mock/` | 仅 `IS_AUTH_MOCK=true` 时使用 | 在生产构建中被引用 |

## 6. 数据流

### 6.1 API 请求

```
页面组件
    ↓ 调用 api/{domain}.ts 函数
api/assets.ts → apiRequest()
    ↓
lib/api.ts → 封装 fetch + envelope 解析
    ↓
或 Axios 直接请求（lib/http.ts）
    ↓
后端 API → JSON → ApiEnvelope { code, message, data }
    ↓
返回业务数据 → setState / Zustand store
```

### 6.2 认证令牌传递

```
request interceptor (lib/http.ts):
  从 localStorage/sessionStorage 读取 access_token
  ↓
  Authorization: Bearer <token>
  ↓
response interceptor:
  401 + AUTH_TOKEN_MISSING/INVALID/EXPIRED → 清除 session → 跳转 /login
```

### 6.3 认证状态管理

```typescript
// stores/authSession.ts (Zustand)
type AuthSessionStore = {
  me: AuthUserProfile | null
  hydrated: boolean
  authLoadError: string | null
  fetchMe: () => Promise<void>
  setMe: (profile: AuthUserProfile) => void
  logout: () => void
}
```

## 7. 认证与授权

| 模块 | 说明 |
|------|------|
| `auth/login.ts` | 登录编排：Mock 走本地校验库，真实走后端 API |
| `auth/RequireAuth.tsx` | 路由守卫：无 token 跳转 /login |
| `auth/permission.ts` | 从后端 `/auth/me` 响应推导权限（`profileFromApiMe`） |
| `auth/routeGuard.ts` | 路由级细粒度权限：页面不可见时跳转 /403 |
| `auth/Auth.tsx` | React 组件级 Auth wrapper（Conditional rendering） |
| `auth/logout.ts` | 清除 token + profile → 跳转 /login |

### Mock vs 真实模式

| 模式 | 触发条件 | 登录方式 |
|------|---------|---------|
| Mock | `VITE_AUTH_MOCK=true` | 本地校验账号密码，生成 `MOCK_TOKEN` |
| 真实 | 缺省/`VITE_AUTH_MOCK=false` | 请求 `/api/v1/auth/login` 获取 JWT |

### 统一身份模式（扩展）

```typescript
// main.tsx — postMessage 监听 H-UMDG 推送
window.addEventListener('message', (event) => {
  if (event.data?.type === 'h-umdg-auth' && event.data?.token) {
    setAccessToken(event.data.token, true)  // 存入 localStorage
    // 页面自动跳转后，RequireAuth 读取 token → 允许放行
  }
})
```

## 8. 复用规则

### 8.1 组件层次

| 层次 | 说明 | 示例 | 复用范围 |
|------|------|------|---------|
| API 函数 | 每个业务域的请求封装 | `api/assets.ts` | 多个页面复用 |
| 选择器 | 通用主数据选择器 | `BusinessPartnerSelector` | 资产建档/维修派工 |
| 布局 | 页面结构和路由出口 | `ProAdminLayout` | 全局 |
| 页面 | 完整业务页面 | `RepairTicketsPage` | 独立路由 |
| 工具 | 纯函数/工具类 | `lib/token.ts` | 全局 |

### 8.2 选择器组件模式

```typescript
// 通用模式：选择器 + 弹出搜索 + 确认选中
<BusinessPartnerSelector
  roleType="供应商"
  value={selectedPartnerId}
  onChange={(partner) => setSelectedPartner(partner)}
/>
```

### 8.3 复用优先级

1. **Ant Design / ProComponents 内置** → 优先使用
2. **H-MELC 内部提取** → 同一组件在 3+ 页面中出现时提取
3. **不跨项目复用** → H-MELC 和 H-UMDG 独立部署


移动端定位：**扫码解析与标签补打 H5**，不是完整移动端应用。主要功能：
- 扫描设备二维码 → 解析资产信息
- 连接蓝牙标签打印机 → 补打资产标签

## 12. 前端安全

| 措施 | 实现 |
|------|------|
| JWT 存储 | localStorage（持久）/ sessionStorage（会话） |
| 自动过期跳转 | Axios 响应拦截器判断 401 + 特定 code |
| 请求自动鉴权 | Axios 请求拦截器注入 `Authorization: Bearer` |
| 路由守卫 | RequireAuth 检查 token 是否存在 |
| 权限控制 | PermissionOutlet + routeGuard 路由级拦截 |
| Mock 隔离 | Mock 数据仅在 `IS_AUTH_MOCK=true` 时使用 |

> **统一身份扩展**：H-UMDG → postMessage → H-MELC 接收 → setAccessToken
> 无密码明文传输，JWT 仅通过 window.postMessage 在相同 origin 间传递。
