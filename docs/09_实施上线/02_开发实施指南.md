## 二、后端（FastAPI）本地运行

```bash
cd backend
python -m venv .venv
# Windows PowerShell: .venv\Scripts\Activate.ps1
pip install -e ".[dev]"
cp .env.example .env    # 按环境修改 DATABASE_URL、密钥等
```

- 数据库迁移：**PostgreSQL** 上执行 **`alembic upgrade head`**（各模块就绪要求见接口返回 **503** 时的说明文案）。
- 运行服务（Windows 若未把 Scripts 加入 PATH，请用 **`python -m`**）：**`python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000`**（先 `pip install -e ".[dev]"` 并激活 `.venv`）。
- **Windows `WinError 10013` 绑定端口失败**：多为 **8000 已被占用**（本机已有另一个 uvicorn/Docker/MkDocs 等）或 **Hyper-V 保留端口段** 命中。处理：① `netstat -ano | findstr :8000` 查看占用 PID，结束多余进程；或 ② 换端口启动，例如 **`--port 8010`**，并在 `frontend-admin` 的 **`.env`** 中设置 **`VITE_API_PROXY_TARGET=http://127.0.0.1:8010`**（走 Vite 代理时）或 **`VITE_API_BASE_URL=http://127.0.0.1:8010`**（直连时）。
- 测试：

```bash
pytest
```

与 **`docs/06`** 对齐的契约用例多见于 **`backend/tests/`** 中文件名含 **`contract`** / **`doc`** 的模块；变更接口时请同步修改或 **`skip`/TODO** 标明缺口。

## 三、前端管理端（frontend-admin）

```bash
cd frontend-admin
cp .env.example .env      # 默认 `VITE_AUTH_MOCK=false`；`VITE_API_BASE_URL` 建议留空以走 Vite `/api` 代理到本机 8000
npm install
npm run dev               # 默认 http://127.0.0.1:5102 ，需后端 CORS 放行
npm run build
```

详见 **`frontend-admin/README.md`**。

## 四、移动扫码端（frontend-mobile）

```bash
cd frontend-mobile
npm install
npm run dev              # 默认 http://0.0.0.0:5175，手机访问 http://<本机局域网IP>:5175
npm run build
```

- 默认 `/api` 由 Vite 代理到 `VITE_API_PROXY_TARGET`，未配置时为 **`http://127.0.0.1:8010`**；如后端运行在 8000，请设置 **`VITE_API_PROXY_TARGET=http://127.0.0.1:8000`**。
- 当前一期能力：登录、扫码/输入 token、读取 `GET /api/v1/assets/label-templates` 选择纸张/尺寸/版式预设、读取 `GET /api/v1/assets/{asset_id}/print-label?template_code=...`、按模板预览、蓝牙/WiFi 连接工作流、手机直连打印桥接、系统打印兜底。
- 手机相机扫码依赖浏览器 `BarcodeDetector` 能力，真机浏览器通常还要求 HTTPS 或 App WebView 安全上下文；不支持时可粘贴二维码内容或 token。真正直连精臣蓝牙/WiFi 打印机需 App / 小程序封装精臣移动 SDK，并向 H5 注入 **`window.JcMobilePrinter.scanPrinters/connectPrinter/printLabel`**。

## 六、知识库原始资料（可选）

仓库内 **`knowledge-base/`** 用于承载制度、案例等 Markdown/YAML 元数据示例，与 **`docs/04_AI设计/`** RAG 设计配套；运行时入库仍以 **`backend`** 迁移与 **`POST /api/v1/knowledge/documents`**（multipart）等业务接口为准。

可维护 **`knowledge-base/metadata/document_index.yaml`** 作为书目索引示意；条目路径与标签规范按数据治理要求扩展。

## 七、建议迭代顺序（与一期范围对齐）

1. **身份与 RBAC**：登录、令牌、`/auth/me` 与后端角色常量一致。  
2. **设备台账**：`GET/PATCH .../assets`、详情聚合 **`GET .../assets/{id}`**。  
3. **报修闭环**：`GET/POST .../repairs` 及工单详情与各 **`POST`** 动作（见 **`docs/06`** 第二章、第三章）。  
4. **知识库**：列表、详情、multipart 上传、占位 **`/knowledge/chat`**。  
5. **横向能力**：审计、工作流、财务/供应商等按里程碑扩展（见 **`docs/01_需求文档/06`** 及各模块文档）。

占位前端路由在接通真实 API 后，须在 **`docs/05/03`** 与（若变更契约）**`docs/06`** 增量登记。
