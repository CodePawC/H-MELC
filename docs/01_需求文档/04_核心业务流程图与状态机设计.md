# 《核心业务流程图与状态机设计 V1.0》
## 一、设备生命周期状态机
```
规划中→ 采购中→ 到货待验收→ 验收中→ 已验收→ 建账赋码→ 安装调试→ 培训完成→ 在用→ 维修中 / 计量中 / 巡检中 / 调拨中→ 闲置 / 停用隔离→ 待报废→ 已报废→ 档案归档
```

关键规则：
| 当前状态 | 可进入状态                    |
|------|--------------------------|
| 在用   | 维修中、计量中、巡检中、调拨中、停用隔离、待报废 |
| 维修中  | 在用、外修中、返厂中、停用隔离          |
| 停用隔离 | 维修中、待报废、在用               |
| 已报废  | 只能归档，不可恢复在用              |


---
## 二、报修维修状态机
```
科室报修→ AI分诊→ 待接单→ 已接单→ 诊断中→ 维修中→ 待测试→ 待科室确认→ 已完成
```

异常分支：
```
诊断中→ 等待备件→ 第三方维修→ 返厂维修→ 待报价→ 待审批→ 维修中
```

不良事件分支：
```
AI识别疑似不良事件→ 人工确认→ 发起不良事件流程→ 维修流程继续或设备停用隔离
```

---
## 三、发票付款状态机
```
供应商上传→ AI识别中→ 待人工核对→ 已匹配业务单据→ 待审核→ 审核通过→ 未付款→ 部分付款→ 已结清
```

异常分支：
```
待人工核对→ 识别失败→ 供应商补充资料待审核→ 驳回→ 供应商修改后重新提交已匹配→ 金额不符→ 进入争议处理
```

---
## 四、付款分摊流程
```
财务登记付款→ 选择供应商→ 选择发票/应付款→ 分摊金额→ 系统校验→ 更新发票已付金额→ 更新未付金额→ 生成付款记录→ 供应商可查看
```

系统校验规则：
| 校验项   | 规则           |
|-------|--------------|
| 分摊总额  | 不得超过付款金额     |
| 发票付款  | 不得超过发票未付金额   |
| 应付款状态 | 结清后不可重复分摊    |
| 付款记录  | 不允许物理删除，只能冲销 |


---
## 五、供应商竞价状态机
```
项目草稿→ 内部审核→ 对外发布→ 供应商报名→ 资质审核→ 报价中→ 截止报价→ 开标/比价→ 审计审核→ 财务审核→ 确认中标→ 执行中→ 验收完成→ 关闭
```

关键控制：
| 节点    | 控制要求        |
|-------|-------------|
| 报价中   | 供应商互不可见     |
| 截止报价后 | 不允许修改报价     |
| 开标/比价 | 自动生成比价表     |
| 审计审核  | 全流程留痕       |
| 财务审核  | 关注付款来源和欠款风险 |
| 中标确认  | 生成业务单据链     |

## 六、AI介入节点总表
| 流程   | AI节点  | 输出              |
|------|-------|-----------------|
| 报修   | 提交后   | 故障分类、风险等级、推荐工程师 |
| 维修   | 诊断中   | 故障树、相似案例、维修建议   |
| 维修完成 | 关闭前   | 维修报告、不良事件提示     |
| 发票上传 | 上传后   | OCR结构化结果        |
| 随货同行 | 上传后   | 批号、数量、注册证识别     |
| 付款管理 | 付款计划前 | 付款优先级建议         |
| 竞价项目 | 发布前   | 需求摘要、技术参数提取     |
| 效益分析 | 定期任务  | ROI、使用率、调拨建议    |
| 知识库  | 流程结束后 | 自动沉淀案例          |
