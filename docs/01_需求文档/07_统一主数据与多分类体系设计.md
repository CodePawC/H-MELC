# 《统一主数据与多分类体系设计（MDM + Taxonomy）》

# Unified Master Data & Multi-Taxonomy Architecture

这部分实际上决定：

- 系统未来是否能扩展
- AI是否真正可用
- 多系统是否能融合
- 数据是否可治理
- 是否能形成医院级数据资产

很多医院系统后期失败，不是业务功能不够，而是：

# 主数据设计崩溃。

---

# 一、总体设计目标

构建：

# “H-UMDG + 医学装备运营平台协同治理”

实现：

|能力|目标|
|---|---|
|数据统一|H-UMDG 管标准口径，医学装备运营平台管设备业务事实|
|多系统共享|HIS、SPD、设备、BI统一|
|多分类体系|一个设备支持多个分类维度|
|审核治理|新型号/新分类可审核|
|AI友好|支持标签化与向量化|
|动态扩展|后期无需频繁改表|
|审计追踪|所有字典变更留痕|
|标准化|支持国标/FHIR/UDI扩展|

# 第三层：业务引用层（Business Reference Layer）

业务系统不替代 H-UMDG 的标准口径治理；医学装备运营平台是具体设备全生命周期业务事实的系统。

| 对象 | 权威边界 |
|---|---|
| 设备业务台账 | H-MELC 负责业务事实与建档；H-UMDG 负责设备分类、标准名称、监管属性和厂商机构标准身份 |
| 科室主数据 | H-UMDG 权威，H-MELC 只引用 |
| 人员主数据 | H-UMDG 权威，H-MELC 可维护工程师技能、值班、资质等业务扩展 |
| 厂商机构主数据 | H-UMDG 权威，H-MELC 可维护联系人、履约评价、设备服务范围等业务扩展 |
| 耗材主数据 | H-UMDG / SPD 权威，H-MELC 只引用 |
| 收费项目、医保编码 | HIS / 医保平台 / H-UMDG 权威，H-MELC 用于效益与 ROI 分析 |

# 三-A、主数据来源配置

医学装备运营 OS 在 **系统设置 / 主数据来源配置** 中维护各主数据对象的数据来源策略。

页面字段：

| 字段 | 说明 |
|---|---|
| 主数据对象 | 科室、人员、设备、耗材、供应商、收费项目、医保编码、设备分类字典、设备状态字典、计量属性字典、风险等级字典 |
| 当前权威来源 | 当前采用的权威系统，如 H-UMDG、医学装备运营平台、HIS、SPD、医保平台 |
| 可选数据来源 | 可选择或组合的数据来源 |
| 同步方式 | API 实时调用、定时同步、只读缓存刷新、候选申请同步 |
| 同步频率 | 实时、分钟级、小时级、每日定时等 |
| 是否允许本地维护 | 是、否、部分允许 |
| 冲突处理策略 | 以 H-MELC 业务事实为准、以 H-UMDG 标准口径为准、保留双方记录、人工审核合并、生成候选申请 |
| 最近同步时间 | 最近一次成功或失败同步时间 |
| 同步状态 | 未同步、同步中、已同步、同步失败、待审核、冲突待处理 |
| 操作 | 查看详情、编辑、立即同步、查看冲突、查看版本 |

推荐默认配置：

| 主数据对象 | 权威来源 | 可选来源 | 同步方式 | 是否允许本地维护 | 冲突处理 |
|---|---|---|---|---|---|
| 设备业务台账 | 医学装备运营平台 | H-MELC、H-UMDG 引用快照、固定资产系统 | API 引用与快照保存 | 是 | 业务事实以 H-MELC 为准，标准口径以 H-UMDG 为准 |
| 科室主数据 | H-UMDG | H-UMDG、HIS、本地维护 | API 实时调用或定时同步 | 否 | 以 H-UMDG 为准 |
| 人员主数据 | H-UMDG | H-UMDG、HIS、人事系统、本地维护 | 定时同步 | 部分允许 | 冲突数据进入待审核 |
| 厂商机构主数据 | H-UMDG | H-UMDG、SPD、财务系统、本地业务快照 | 定时同步 | 部分允许 | 人工审核 |
| 耗材主数据 | H-UMDG / SPD | H-UMDG、SPD、医保平台、本地维护 | 定时同步 | 否 | 以 H-UMDG 标准目录为准 |
| 收费项目主数据 | HIS / H-UMDG | HIS、医保平台、H-UMDG | 定时同步 | 否 | 以 HIS 和医保映射结果为准 |
| 医保编码 | 医保平台 / H-UMDG | 医保平台、H-UMDG | 定时同步 | 否 | 以医保平台为准 |

设备新增时，医学装备运营平台调用 H-UMDG 获取设备分类、设备标准名称、医疗器械分类目录监管信息和厂商机构标准身份；H-MELC 只保存 H-UMDG 返回的引用字段和必要快照。找不到标准名称或厂商机构时提交候选申请，后续由 H-UMDG 审核发布。

设备不允许物理删除。删除类操作必须改为状态变化：作废、停用、待报废、已报废、归档、隐藏；所有状态变化保留操作日志、审批记录和版本记录。

关键字段修改需要进入 H-MELC 审核流程；涉及标准分类、标准名称、监管属性和厂商机构身份时，只更新 H-UMDG 引用字段和快照，不在 H-MELC 内维护权威字典。普通字段如安装位置、联系人、使用备注、附件说明、维护说明、标签备注可直接保存。

# 三、多分类体系（核心设计）

这是系统最关键部分之一。

---

# 1. 为什么不能单分类

传统：

```
设备分类 = CT
```

完全不够。

因为：

CT同时属于：

- 医学影像
- 放射设备
- 大型设备
- 强监管设备
- DICOM设备
- 高价值设备
- 高耗电设备
- 高维修成本设备

所以：

# 必须采用多分类模型。
