# 《数据库核心表结构设计 V1.0》
下面先设计“可开发的一期核心表”，重点覆盖：
  * 设备台账
  * 一机一码
  * 生命周期事件
  * 报修维修

  * 知识库
  * 审计日志


---
# 一、数据库 Schema 划分
建议 PostgreSQL 使用多 schema：
```
integration -- 外部系统只读缓存与候选申请
asset -- 设备资产
repair -- 报修维修
supplier -- 供应商门户
finance -- 发票付款
workflow -- 工作流
ai -- AI分析
knowledge -- 知识库
audit -- 审计日志
```

H-UMDG 是另一个系统，负责基础主数据和标准字典权威维护。H-MELC 仅保存 H-UMDG 引用字段、必要快照、只读缓存和候选申请，不维护 H-UMDG 权威字典。

## 1a\. asset.equipment_classification_impact 分类变更影响表

H-UMDG 分类目录变更后，H-MELC 不自动覆盖设备档案，而是生成影响提醒和复核任务。

```sql
CREATE TABLE asset.equipment_classification_impact (
  impact_id UUID PRIMARY KEY,
  equipment_id UUID NOT NULL REFERENCES asset.asset(id),
  old_classification_id VARCHAR(128),
  old_classification_code VARCHAR(128),
  new_classification_id VARCHAR(128),
  change_type VARCHAR(64) NOT NULL,
  impact_level VARCHAR(16) NOT NULL,
  impact_reason VARCHAR(500) NOT NULL,
  source_change_id VARCHAR(128) NOT NULL,
  status VARCHAR(32) DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT now(),
  handled_by VARCHAR(128),
  handled_at TIMESTAMP
);
```

`impact_level`：`low`、`medium`、`high`；`status`：`pending`、`confirmed`、`adjusted`、`ignored`。

# 三、统一报修中心与维修报修核心表

统一报修中心把“多渠道原始消息”和“正式维修工单”拆开建模：`unified_repair_message` 记录电脑端、移动端、设备二维码、微信、企业微信、飞书、钉钉、AI聊天框、语音、图片、人工代录、系统预警等入口的原始消息和 AI 抽取结果；设备匹配和确认完成后才生成 `repair_order`。`repair_order` 是统一报修中心输出的标准工单结果，不等同于整个报修功能。

## 1\. repair.repair_order 维修工单表
```
CREATE TABLE repair.repair_order ( id UUID PRIMARY KEY, order_code VARCHAR(64) UNIQUE NOT NULL, source_message_id UUID, source_channel VARCHAR(64), asset_id UUID NOT NULL REFERENCES asset.asset(id), report_department_id UUID, reporter_id UUID, reporter_name VARCHAR(128), reporter_phone VARCHAR(64), fault_description TEXT, fault_type VARCHAR(64), fault_level VARCHAR(32), priority VARCHAR(32), affects_clinical_use BOOLEAN DEFAULT FALSE, involves_patient BOOLEAN DEFAULT FALSE, is_emergency_device BOOLEAN DEFAULT FALSE, is_life_support_device BOOLEAN DEFAULT FALSE, order_status VARCHAR(64) NOT NULL, assigned_engineer_id UUID, accepted_at TIMESTAMP, completed_at TIMESTAMP, confirmed_at TIMESTAMP, is_outsourced BOOLEAN DEFAULT FALSE, is_return_factory BOOLEAN DEFAULT FALSE, is_chargeable BOOLEAN DEFAULT FALSE, estimated_cost NUMERIC(14,2), actual_cost NUMERIC(14,2), ai_extract_result_id UUID, ai_risk_level VARCHAR(64), ai_incident_suggestion BOOLEAN DEFAULT FALSE, created_at TIMESTAMP DEFAULT now(), updated_at TIMESTAMP DEFAULT now());
```

说明：`source_message_id` 关联 `repair.unified_repair_message.id`，用于追溯该工单来自哪个渠道消息；设备二维码入口生成工单时同样只记录设备身份来源，不改变二维码归属资产中心的边界。

# 九、H-UMDG 外部接入缓存与申请表

H-UMDG 是医院统一主数据治理平台；以下表属于 H-MELC 侧集成表，只用于只读缓存、降级和候选申请，不构成权威主数据。

## 1\. integration.hmdm_dictionary_cache

```sql
CREATE TABLE integration.hmdm_dictionary_cache (
  id UUID PRIMARY KEY,
  source_type VARCHAR(64) NOT NULL,
  source_code VARCHAR(128) NOT NULL,
  source_name VARCHAR(255),
  payload_json JSONB NOT NULL,
  synced_at TIMESTAMP NOT NULL,
  expire_at TIMESTAMP NOT NULL,
  status VARCHAR(32) DEFAULT 'ACTIVE',
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now(),
  UNIQUE (source_type, source_code)
);
```

`source_type` 取值：`equipment_category`、`equipment_standard_name`、`device_classification`、`manufacturer_vendor`。缓存用途是提升页面加载速度、H-UMDG/H-UMDG 暂不可用时提供只读备选、保留最近一次成功同步数据。缓存过期时必须提示，不允许用户直接编辑。

## 1a\. integration.hmdm_classification_change

```sql
CREATE TABLE integration.hmdm_classification_change (
  change_id VARCHAR(128) PRIMARY KEY,
  classification_id VARCHAR(128),
  classification_code VARCHAR(128),
  classification_name VARCHAR(255),
  version_id VARCHAR(128),
  change_type VARCHAR(64) NOT NULL,
  change_reason TEXT,
  old_payload JSONB,
  new_payload JSONB,
  target_classification_id VARCHAR(128),
  target_classification_code VARCHAR(128),
  occurred_at TIMESTAMP DEFAULT now(),
  created_at TIMESTAMP DEFAULT now()
);
```

该表保存从 H-UMDG 医疗器械分类目录拉取的增量变更事件，用于生成设备档案分类影响提醒；不保存或维护分类目录权威明细。

## 2\. integration.equipment_standard_name_request

```sql
CREATE TABLE integration.equipment_standard_name_request (
  id UUID PRIMARY KEY,
  proposed_name VARCHAR(255) NOT NULL,
  alias_names JSONB,
  suggested_category VARCHAR(255),
  reason TEXT NOT NULL,
```

找不到设备标准名称时，H-MELC 只提交候选申请，后续同步给 H-UMDG 审核，不写入本系统设备标准名称权威库。
