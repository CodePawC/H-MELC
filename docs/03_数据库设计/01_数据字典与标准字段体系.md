# 一、数据字典总体设计原则
## 核心原则
### 1\. 全局统一命名
禁止：
```
device_idequipment_idassetid
```

同时存在。
必须统一。
---
### 2\. 所有枚举必须字典化
禁止：
```
status = "维修中"status = "repairing"status = "REPAIR"
```

混用。
必须：
```
REPAIRING
```

统一编码。
---
### 3\. 所有字段必须：
| 属性    | 必须 |
|-------|----|
| 中文名   | 是  |
| 英文名   | 是  |
| 类型    | 是  |
| 长度    | 是  |
| 是否必填  | 是  |
| 默认值   | 是  |
| 字典来源  | 是  |
| 是否可扩展 | 是  |
| 是否审计  | 是  |


---
# 二、统一命名规范
建议：
# snake_case
---
## 表名
```
asset.assetrepair.repair_orderfinance.invoice
```

---
## 字段名
```
asset_codecreated_atupdated_atdepartment_id
```

---
## Boolean字段
必须：
```
is_activeis_deletedis_outsourced
```

不要：
```
active_flagdeleted
```
