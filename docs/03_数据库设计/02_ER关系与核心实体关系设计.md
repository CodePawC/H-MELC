# ER关系图与核心实体关系设计 V1.0

# 《ER关系图与核心实体关系设计 V1.0》
## 一、核心实体分组
系统核心实体分为 9 大类：
| 实体域   | 核心实体                    |
|-------|-------------------------|
| 主数据域  | 科室、位置、厂家、供应商、型号、分类      |
| 设备资产域 | 设备、二维码、生命周期事件、设备档案      |
| 维修域   | 报修工单、维修记录、维修报告、附件       |
| PM巡检域 | PM计划、PM任务、巡检任务、巡检记录     |
| 合规模块  | 计量记录、证照、检测报告、风险预警       |
| 供应商域  | 供应商、供应商账号、资质、竞价项目、报价    |
| 财务域   | 发票、随货同行、应付款、付款记录、付款分摊   |
| AI域   | AI任务、AI结果、AI审核、Prompt模板 |
| 知识库域  | 知识文档、切片、向量、反馈           |

# 三、设备资产核心关系
## 1\. 设备与主数据
```
asset.asset N:1 mdm.department N:1 mdm.location N:1 mdm.manufacturer N:1 mdm.supplier N:1 mdm.equipment_model
```

说明：
  * 一台设备归属一个当前科室
  * 一台设备有一个当前物理位置
  * 一台设备对应一个型号
  * 一台设备可关联厂家与供应商
  * 历史调拨通过调拨记录保存，不直接覆盖历史


---
## 2\. 设备与生命周期事件
```
asset.asset 1:N asset.asset_eventasset.asset 1:N asset.asset_status_history
```

说明：
  * 所有设备行为都写入 asset_event
  * 状态变化单独写入 asset_status_history
  * 事件表用于追溯、AI分析、审计


---
## 3\. 设备与分类
```
asset.asset N:N mdm.taxonomy_node
```

中间表：
```
asset.asset_taxonomy_relation
```

说明：
一台设备可以同时属于：
  * 医学影像设备
  * 放射设备
  * 强检设备
  * 高风险设备
  * 大型设备
  * DICOM设备


---
# 四、维修工单关系
## 1\. 设备与维修
```
asset.asset 1:N repair.repair_order
```

说明：
一台设备可产生多张维修工单。
---
## 2\. 维修工单内部关系
```
repair.repair_order 1:N repair.repair_attachment 1:N repair.repair_process_record 1:1 repair.repair_report 1:N ai.ai_analysis_result
```

说明：
  * 一个工单可以有多个附件
  * 一个工单可以有多个过程记录
  * 一个工单最终形成一份维修报告
  * 一个工单可以触发多次AI分析


---
## 3\. 工单与供应商竞价
```
repair.repair_order 1:N supplier.supplier_projectsupplier.supplier_project 1:N supplier.supplier_bidsupplier.supplier 1:N supplier.supplier_bid
```

说明：
  * 一个维修工单可发起一个或多个外修项目
  * 一个项目可由多个供应商报价
  * 供应商之间报价隔离
  * 中标后形成维修执行关系


---
# 五、供应商与财务关系
## 1\. 供应商基础关系
```
supplier.supplier 1:N supplier.supplier_user 1:N supplier.supplier_qualification 1:N finance.invoice 1:N finance.delivery_document 1:N finance.payable 1:N finance.payment_record
```

说明：
一个供应商下可以有多个账号、多个资质、多张发票、多条付款记录。
---
## 2\. 发票与业务对象关系
```
finance.invoice N:1 supplier.supplier N:1 repair.repair_order 或 purchase_order 或 consumable_order
```

建议字段：
```
related_business_typerelated_business_id
```

说明：
发票可能对应：
  * 设备维修工单
  * 耗材采购订单
  * 设备采购合同
  * 外修项目
  * 其他付款事项


因此不建议写死外键，而采用业务对象引用模式。
---
## 3\. 应付款关系
```
finance.payable N:1 supplier.supplier N:1 source_business_object
```

说明：
应付款是业务验收后形成的财务义务。
来源可以是：
  * 验收完成的维修工单
  * 已入库耗材
  * 已验收设备
  * 合同阶段付款


---
## 4\. 付款分摊关系
```
finance.payment_record N:N finance.invoicefinance.payment_record N:N finance.payable
```

中间表：
```
finance.payment_allocation
```

这是财务模块最关键关系。
支持：
  * 一张发票多次付款
  * 一次付款覆盖多张发票
  * 一笔付款同时分摊多个应付款
  * 后续冲销和调整


---
# 六、AI实体关系
## 1\. AI任务与业务对象
```
ai.ai_task N:1 source_business_object
```

通过：
```
source_modulesource_object_id
```

关联业务对象。
---
## 2\. AI结果关系
```
ai.ai_task 1:N ai.ai_analysis_resultai.ai_analysis_result N:1 human_review_user
```

说明：
  * 一个AI任务可以产生多个结果版本
  * AI结果必须可审核
  * 采用后要记录人工确认人


---
## 3\. AI与知识库
```
knowledge.knowledge_document 1:N knowledge.knowledge_chunkknowledge.knowledge_chunk 1:1 knowledge.knowledge_embeddingai.ai_task N:N knowledge.knowledge_chunk
```

建议增加：
```
ai.ai_retrieval_reference
```

用于记录AI回答引用了哪些知识片段。
---
# 七、知识库关系
```
knowledge.knowledge_document 1:N knowledge.knowledge_chunk 1:N knowledge.knowledge_feedback
```

知识来源可以是：
  * 维修报告
  * 厂家说明书
  * 制度文件
  * 不良事件报告
  * 供应商资料
  * 财务对账规则
  * 计量证书
  * 放射检测报告


---
# 八、工作流关系
```
workflow.workflow_definition 1:N workflow.workflow_nodeworkflow.workflow_definition 1:N workflow.workflow_transitionworkflow.workflow_instance N:1 workflow.workflow_definitionworkflow.workflow_instance 1:N workflow.workflow_taskworkflow.workflow_task 1:N workflow.workflow_action_log
```

业务对象通过：
```
business_typebusiness_id
```

绑定工作流实例。
适用场景：
  * 外修审批
  * 竞价审核
  * 报废审批
  * 付款审核
  * 不良事件流程
  * 型号主数据审核


---
# 九、审计关系
```
audit.audit_log N:1 user N:1 business_object
```

通过：
```
object_typeobject_id
```

关联业务对象。
所有关键表建议开启审计：
  * asset.asset
  * repair.repair_order
  * supplier.supplier_bid
  * finance.invoice
  * finance.payment_record
  * workflow.workflow_task
  * ai.ai_analysis_result

# 十、核心关系总结
| 关系        | 类型  | 说明       |
|-----------|-----|----------|
| 设备-维修工单   | 1:N | 一台设备多次维修 |
| 设备-生命周期事件 | 1:N | 全过程追溯    |
| 设备-分类标签   | N:N | 多分类体系    |
| 工单-维修记录   | 1:N | 过程记录     |
| 工单-维修报告   | 1:1 | 闭环报告     |
| 工单-AI结果   | 1:N | 多次AI辅助   |
| 供应商-发票    | 1:N | 多张发票     |
| 供应商-付款    | 1:N | 多次付款     |
| 付款-发票     | N:N | 分摊模型     |
| 付款-应付款    | N:N | 分摊模型     |
| 文档-知识切片   | 1:N | RAG基础    |
| AI任务-知识切片 | N:N | 引用追溯     |
