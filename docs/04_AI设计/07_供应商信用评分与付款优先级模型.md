# 《供应商信用评分与付款优先级模型 V1.0》
## 一、模块定位
这个模块非常关键。
因为医院现实情况是：
  * 付款周期长
  * 发票积压
  * 部分付款频繁
  * 外修与耗材依赖高
  * 高风险供应商容易停供/拒修
  * 财务、设备科、审计之间信息不透明


因此系统必须建立：
# “供应商画像 + 付款优先级模型”
实现：
  * 付款透明化
  * 风险预警
  * 供应链稳定
  * 医疗安全保障
  * 领导决策支持

# 十、付款优先级模型（核心）
付款优先级：
# 不等于信用评分。
因为：
  * 有些供应商虽然一般，但临床依赖极高。
  * 有些供应商金额不大，但影响ICU运行。

# 十七、数据库设计建议
## supplier_credit_score
```
CREATE TABLE supplier.supplier_credit_score ( id UUID PRIMARY KEY, supplier_id UUID NOT NULL, performance_score NUMERIC(5,2), response_score NUMERIC(5,2), quality_score NUMERIC(5,2), compliance_score NUMERIC(5,2), finance_score NUMERIC(5,2), complaint_score NUMERIC(5,2), total_score NUMERIC(5,2), risk_level VARCHAR(32), calculated_at TIMESTAMP DEFAULT now());
```
