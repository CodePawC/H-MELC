# 医疗设备ROI算法模型设计 V1.0

# 《医疗设备ROI算法模型设计 V1.0》
## 一、ROI分析目标
ROI模块不只是算“赚不赚钱”，而是服务于医院设备配置决策：
  * 是否值得继续使用
  * 是否需要调拨
  * 是否需要新增采购
  * 是否维修成本过高
  * 是否进入更新报废周期
  * 是否存在低效闲置
  * 是否存在科室配置不足


---
# 二、核心指标体系
| 指标     | 含义                 |
|--------|--------------------|
| 使用率    | 实际使用时间 / 可用时间      |
| 开机率    | 开机时间 / 应开机时间       |
| 收入贡献   | 设备相关医疗收入           |
| 维修成本   | 维修费用 + 配件费用 + 外修费用 |
| 停机损失   | 停机期间估算收入损失         |
| 折旧成本   | 按年限分摊设备原值          |
| 单次服务成本 | 总成本 / 服务人次         |
| ROI    | 净收益 / 投入成本         |
| 投资回收期  | 原值 / 年净收益          |


---
# 三、基础数据来源
| 数据     | 来源           |
|--------|--------------|
| 设备原值   | 设备台账         |
| 折旧年限   | 财务/资产数据      |
| 检查人次   | HIS/PACS/LIS |
| 收入金额   | HIS/收费系统     |
| 开机时间   | 设备接口/人工登记    |
| 维修费用   | 维修工单         |
| 停机时间   | 工单状态         |
| 耗材成本   | SPD/耗材系统     |
| 科室借用频次 | 应急调配模块       |


---
# 四、使用率计算
```
使用率 = 实际服务时间 / 可用工作时间 × 100%
```

如果没有设备接口数据，可先用替代口径：
```
使用率 = 实际检查/治疗人次 ÷ 理论最大服务能力 × 100%
```

示例：
```
CT每日理论最大检查量 = 80人次实际日均检查量 = 48人次使用率 = 48 / 80 = 60%
```

---
# 五、收入贡献
```
收入贡献 = Σ 设备相关收费项目收入
```

需要与医院集成平台对接：
  * 收费项目编码
  * 检查项目编码
  * 设备编码
  * 科室编码


如果系统初期无法精确绑定设备，可先按：
```
科室 + 项目 + 设备类型
```

进行估算。
---
# 六、成本模型
## 1\. 年折旧成本
```
年折旧成本 = 设备原值 / 折旧年限
```

## 2\. 维修成本
```
维修成本 = 院内维修成本 + 外修费用 + 配件费用 + 返厂费用
```

## 3\. 停机损失
```
停机损失 = 停机小时数 × 平均小时收入
```

## 4\. 综合成本
```
综合成本 = 折旧成本 + 维修成本 + 耗材成本 + 停机损失
```

---
# 七、ROI计算
```
ROI = （收入贡献 - 综合成本） / 综合成本 × 100%
```

也可以采用投资视角：
```
投资ROI = 年净收益 / 设备原值 × 100%
```

建议系统同时保留两个口径：
| 口径    | 用途       |
|-------|----------|
| 运营ROI | 判断当前运行效益 |
| 投资ROI | 判断购置投资回收 |


---
# 八、投资回收期
```
投资回收期 = 设备原值 / 年净收益
```

规则：
| 回收期  | 判断    |
|------|-------|
| ≤3年  | 效益较好  |
| 3-5年 | 正常    |
| 5-8年 | 偏低    |
| >8年  | 需重点分析 |


---
# 九、维修成本率
```
维修成本率 = 年维修成本 / 设备原值 × 100%
```

建议阈值：
| 维修成本率  | 判断     |
|--------|--------|
| <3%    | 正常     |
| 3%-8%  | 关注     |
| 8%-15% | 高风险    |
| >15%   | 建议更新评估 |


---
# 十、停机率
```
停机率 = 停机时间 / 统计周期总时间 × 100%
```

用于判断：
  * 设备可靠性
  * 厂家服务能力
  * 是否影响临床
  * 是否需要备用机


---
# 十一、综合效益评分
建议建立 100 分模型：
| 指标     | 权重 |
|--------|----|
| 使用率    | 25 |
| ROI    | 25 |
| 维修成本率  | 20 |
| 停机率    | 15 |
| 科室需求强度 | 10 |
| 监管风险   | 5  |


系统生成：
```
综合效益评分 = 各指标标准化得分 × 权重
```

---
# 十二、AI调拨建议模型
当满足以下条件时触发：
| 条件       | 说明         |
|----------|------------|
| 使用率低     | 本科室长期低于30% |
| 其他科室借用频繁 | 同类设备借用次数高  |
| 设备状态良好   | 健康评分较高     |
| 科室业务量下降  | HIS数据支持    |


AI建议：
```
建议将该设备纳入共享设备池或调拨至需求更高科室。
```

---
# 十三、AI新增采购建议模型
当满足：
| 条件      | 说明           |
|---------|--------------|
| 某科室借用频繁 | 同类设备月借用 ≥ 3次 |
| 现有设备高负荷 | 使用率 ≥ 85%    |
| 故障影响明显  | 停机导致临床延误     |
| ROI较好   | 同类设备收益良好     |


AI建议：
```
建议评估该科室新增配置同类设备。
```

---
# 十四、AI更新报废建议模型
当满足：
| 条件          | 说明 |
|-------------|----|
| 使用年限接近折旧年限  |    |
| 维修成本率 > 15% |    |
| 故障频次明显升高    |    |
| 停机率高        |    |
| 配件停产        |    |
| 监管风险升高      |    |


AI建议进入：
```
更新论证流程 / 报废评估流程
```

# 十五、ROI快照表设计
建议不要实时计算所有指标。
建立：
```
asset_roi_snapshot
```

字段：
| 字段                | 说明     |
|-------------------|--------|
| asset_id          | 设备     |
| period_type       | 月/季度/年 |
| period_start      | 开始日期   |
| period_end        | 结束日期   |
| usage_rate        | 使用率    |
| revenue_amount    | 收入     |
| repair_cost       | 维修成本   |
| downtime_hours    | 停机小时   |
| depreciation_cost | 折旧     |
| roi               | ROI    |
| payback_period    | 回收期    |
| benefit_score     | 综合评分   |
| ai_recommendation | AI建议   |
